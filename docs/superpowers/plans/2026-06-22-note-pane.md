# Note Pane Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a plain-text Note pane for quick jotting, persisted with the session like every other pane.

**Architecture:** A new `Block::Note { state: NoteState }` variant where `NoteState` wraps iced's `text_editor::Content` (the live editable buffer). The serializable form is a plain `String` in a new `BlockState::Note { content }`. A sidebar button creates one; the block‚Üíview match renders it with `text_editor`; session capture/restore round-trips the text.

**Tech Stack:** Rust (edition 2021), iced 0.14 (`text_editor` widget), serde (session).

## Global Constraints

- **Session-scoped persistence only:** content lives in the pane, captured to `BlockState::Note { content: String }` in `session.json`, restored on launch. No separate notes store/manager.
- **`NoteState::text()` must strip exactly one trailing `\n`** (iced's `Content::text()` appends one). This keeps capture‚Üírestore idempotent (no newline accretion across sessions).
- **Plain text only.** Non-goals: markdown preview, notes list/browser, export, per-note titles beyond the existing double-click pane rename, AI integration.
- `Block` is intentionally NOT `Clone` (holds PTY handles) ‚Äî storing the non-`Clone` `text_editor::Content` is fine.
- Follow existing patterns: new `Block` variants add an arm to every explicit `match self` in `block.rs`; sidebar buttons follow the `sidebar_svg_button` + `with_tooltip` pattern.

//! Note pane state ‚Äî a plain-text scratch buffer for quick jotting.
//!
//! Wraps iced's stateful `text_editor::Content` (the live editable buffer,
//! which must live in the block's state, not be rebuilt each frame). The
//! serializable form is the plain `String` returned by `text()`.

use iced::widget::text_editor;

/// State for a single note pane.
pub struct NoteState {
    /// The live, editable text buffer.
    pub content: text_editor::Content,
}

impl NoteState {
    /// An empty note.
    pub fn new() -> Self {
        Self { content: text_editor::Content::new() }
    }

    /// A note seeded with existing text (used on session restore).
    pub fn with_text(text: &str) -> Self {
        Self { content: text_editor::Content::with_text(text) }
    }

    /// Apply an editor action (typing, selection, cursor movement, etc.).
    pub fn perform(&mut self, action: text_editor::Action) {
        self.content.perform(action);
    }

    /// The current plain-text content, used for session capture.
    ///
    /// iced's `Content::text()` appends a trailing newline; strip exactly one
    /// so capture‚Üírestore is idempotent (no newline accretion across sessions).
    pub fn text(&self) -> String {
        let t = self.content.text();
        t.strip_suffix('\n').map(str::to_string).unwrap_or(t)
    }
}

impl Default for NoteState {
    fn default() -> Self {
        Self::new()
    }
}

#[cfg(test)]
mod tests {
    use super::*;

    #[test]
    fn empty_note_has_empty_text() {
        assert_eq!(NoteState::new().text(), "");
    }

    #[test]
    fn with_text_round_trips_exactly() {
        assert_eq!(NoteState::with_text("hello world").text(), "hello world");
    }

    #[test]
    fn multiline_round_trips_without_newline_accretion() {
        let once = NoteState::with_text("multi\nline").text();
        assert_eq!(once, "multi\nline");
        // Re-seeding the captured text and reading again must be stable.
        let twice = NoteState::with_text(&once).text();
        assert_eq!(twice, "multi\nline");
    }
}
```

// ---------------------------------------------------------------------------
// Note view

/// Build the note pane view: a full-height plain-text editor.
fn note_view<'a>(
    pane: pane_grid::Pane,
    state: &'a workspace::NoteState,
) -> Element<'a, Message> {
    iced::widget::text_editor(&state.content)
        .on_action(move |action| Message::NoteEdited(pane, action))
        .height(Length::Fill)
        .padding(8)
        .into()
}
```

> Uses the fully-qualified `iced::widget::text_editor` so no import changes are needed. `Length` and `Element` are already in scope in this file.
